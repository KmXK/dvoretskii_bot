# Казино — Экономика, веса, оформление

## Валюта

**Обезьянки** 🐵 — внутриигровая валюта казино-раздела.

- Начальный баланс нового игрока: **100 обезьянок**
- Баланс хранится в БД на сервере (поле `User.monkeys`)
- Баланс отображается вверху экрана казино и в команде `/me`

## Дневной бонус

| Параметр | Значение |
|---|---|
| Размер бонуса | 50 🐵 |
| Кулдаун | 24 часа |
| Механика | Нажатие кнопки (НЕ автоматически) |
| Хранение | `User.casino_last_bonus` timestamp в БД |

50 обезьянок = 5 спинов на автомате. Достаточно для короткой сессии, мотивирует возвращаться каждый день.

|---|---|---|

### Математика RTP

```
P(triple 🍒) = 0.30³ = 0.027     →  EV = 2.16
P(triple 🍋) = 0.25³ = 0.01563   →  EV = 1.875
P(triple 🔔) = 0.20³ = 0.008     →  EV = 1.6
P(triple ⭐) = 0.13³ = 0.0022    →  EV = 0.77
P(triple 💎) = 0.08³ = 0.000512  →  EV = 0.358
P(triple 7️⃣) = 0.04³ = 0.000064  →  EV = 0.096

Σ EV(triples) = 6.86

P(любая пара, без тройки) ≈ 0.49  →  EV = 2.46

Общий EV = 6.86 + 2.46 = 9.32
RTP = 9.32 / 10 = 93.2%
```

RTP ~93% — щедро для социального казино. Игрок с дневным бонусом (50 🐵) теряет в среднем ~3.5 за сессию, баланс копится. Пары выпадают часто (~49% спинов), тройки — ~5.4%.

## Ракетка 🚀 (Crash Game)

Краш-игра с общим стримом раундов для всех игроков.

### Механика

- Раунды идут по расписанию (15 сек цикл), привязаны к UTC-часам
- Все игроки видят одну и ту же ракету (shared state через seed + round number)
- Ставка выбирается игроком: 5, 10, 25, 50 🐵
- Фазы раунда: **Ставки (3с)** → **Полёт (до 9с)** → **Результат (3с после краша)**
- Множитель растёт экспоненциально: `mult = e^(0.0005 × t_ms)`
- Игрок должен нажать «Забрать» до краша
- Если забрал — выигрыш = ставка × множитель
- Если не успел — ставка сгорает

### Генерация crash point

```
hash_check = cyrb53(seed + ":" + round + ":c")
if hash_check % 25 == 0:    → crash = 1.00x (мгновенный краш, ~4%)

hash = cyrb53(seed + ":" + round)
u = hash / 2^53
crash_point = min(100, max(1.01, floor(1 / (1-u) × 100) / 100))
```

- Seed генерируется сервером (`HMAC(SHA256, bot_token, period)`)
- Seed ротируется каждые 4 часа
- Все клиенты с одним seed получают одинаковые crash points

### Математика RTP

```
~4% раундов — мгновенный краш (x1.00), ставка сгорает
96% раундов — crash_point = 1/(1-U), U uniform

P(выжить до множителя m) = 0.96 / m
EV при кешауте на m: 0.96/m × m = 0.96
RTP = 96%
```

### Скрытие latency

| Момент | Решение |
|---|---|
| Вход в игру | Анимация «Подключение к орбите» (1.5с мин) скрывает fetch seed |
| Cashout | Optimistic UI — мгновенный отклик, сервер уведомляется фоном |
| Отправка результата | Во время betting-фазы следующего раунда (5 сек буфер) |
| Синхронизация | `serverTime - Date.now()` offset при init |

### Характеристики

| Параметр | Значение |
|---|---|
| RTP | ~96% |
| House edge | ~4% (мгновенные краши) |
| Макс. множитель | ×100 |
| Макс. ставка | 50 🐵 |
| Макс. выигрыш | 5000 🐵 |
| Цикл раунда | переменный (betting + flight + 3с post-crash) |
| Фаза ставок | 3 сек |
| Post-crash | 3 сек |
| Макс. полёт | 9 сек |

## Скачки обезьянок 🏁

Мультиплеерные скачки с синхронизированными забегами. Все игроки видят одни и те же раунды и ставки друг друга.

### Обезьянки-участники

| # | Имя | Эмодзи | Вес | Вероятность | Множитель |
|---|---|---|---|---|---|
| 0 | Бананчик | 🍌 | 30 | 30% | ×2.8 |
| 1 | Кокос | 🥥 | 25 | 25% | ×3.4 |
| 2 | Шимпа | 🐒 | 20 | 20% | ×4.2 |
| 3 | Горилла | 🦍 | 13 | 13% | ×6.5 |
| 4 | Мандарин | 🍊 | 8 | 8% | ×10 |
| 5 | Кинг-Конг | 👑 | 4 | 4% | ×20 |

### Тайминг раунда (22 секунды)

| Фаза | Длительность | Описание |
|---|---|---|
| Ставки | 12с | Игрок выбирает обезьянку и размер ставки |
| Забег | 5с | Анимация гонки, победитель предопределён |
| Результат | 5с | Показ победителя, выплаты, обратный отсчёт |

### Синхронизация

- Раунды привязаны к UTC-часам (как ракетка): `round = floor(elapsed / 22s)`
- Seed = `SHA256("race:" + BOT_TOKEN + ":" + period)`, ротация каждые 4 часа
- Победитель детерминирован: `cyrb53(seed + ":" + round + ":race") % 100` → weighted selection
- Все клиенты вычисляют одного победителя

### Мультиплеер

- Ставки хранятся на сервере (`_race_bets`)
- Клиенты поллят `GET /race/bets` каждые 2.5с во время фазы ставок
- Все игроки видят ставки друг друга в реальном времени
- Одна ставка на игрока за раунд

### Математика RTP

```
EV(Бананчик) = 0.30 × 2.8 = 0.84
EV(Кокос)    = 0.25 × 3.4 = 0.85
EV(Шимпа)    = 0.20 × 4.2 = 0.84
EV(Горилла)  = 0.13 × 6.5 = 0.845
EV(Мандарин) = 0.08 × 10  = 0.80
EV(Кинг-Конг)= 0.04 × 20  = 0.80

RTP ≈ 80–85% (зависит от выбора обезьянки)
```

### Settlement

Серверный (не клиентский). Сервер списывает баланс при размещении ставки и кредитует выигрыш при lazy settlement (при следующем запросе к race-эндпоинтам).

### API

| Метод | Эндпоинт | Описание |
|---|---|---|
| `GET` | `/api/casino/race/init` | Seed + serverTime + balance |
| `POST` | `/api/casino/race/bet` | `{monkeyIdx, amount}` → валидация → списание → `{ok, bets}` |
| `GET` | `/api/casino/race/bets` | `{round, phase, bets, monkeys}` + lazy settlement |

## Шахматы и шашки ♟️

Онлайн-комнаты для шахмат и шашек с режимами:

- Игрок vs игрок
- Игрок vs бот
- Наблюдатели (spectators)
- Ставки зрителей на победителя

### Механика

- Комната создаётся с типом игры: `chess` или `checkers`
- Белые/чёрные — отдельные слоты игроков, остальные подключаются как зрители
- В обеих играх первыми всегда ходят белые
- Опционально включается бот на одной из сторон
- Сложность бота: `easy` / `medium` / `hard`
- Бот может играть как за белых, так и за чёрных (матч корректно стартует для обеих сторон)
- Доска автоматически поворачивается для игрока: свои фигуры всегда снизу (для зрителей — стандартный вид)
- На поле отображаются координаты (`a-h`, `1-8`) с учетом поворота доски
- Подсветка клеток (выбранная фигура, доступные ходы, последний ход) рисуется внутри клетки и не выходит за границы поля
- Для светлых шахматных фигур усилен контраст на светлых клетках
- При превращении пешки доступен выбор фигуры (`Q/R/B/N`) на клиенте
- Зритель может сделать 1 ставку на `white` или `black` до завершения матча
- Выплата по ставке зрителя: `x1.9`
- Можно задать ставку игроков (`stake`) при старте матча:
  - оба игрока вносят одинаковую сумму
  - победитель получает банк
  - при ничьей деньги возвращаются

### Шашки: правила

- Простая шашка ходит вперёд по диагонали на 1 клетку
- Бить можно вперёд и назад
- Поддержано обязательное и множественное взятие (несколько ударов подряд за один ход)
- При обязательном взятии можно выбрать только шашку, у которой есть легальный удар (исключены "псевдо-зависания" из-за клика по заблокированной шашке)
- Дамка ходит по диагонали на любое число пустых клеток
- Дамка бьёт через фигуру соперника с приземлением на любую свободную клетку за ней
- Множественное взятие обрабатывается единым серверным методом и одинаково работает для игрока и бота

### Технические обновления (boardgames)

- В `hard`-режиме шахматного бота используется поиск глубины 2 с `alpha-beta` (negamax) и константами мата
- Входящие WebSocket payload валидируются на сервере (`TypedDict` + проверки формата) для снижения числа «тихих» ошибок
- Неактивные/закрытые WebSocket-подключения удаляются из комнат и лобби при ошибках `send_str`
- Лимит комнат фиксирован как `MAX_ROOMS` без переполнения на +1

### Приглашения

- Поддержаны приглашения в чаты по аналогии с покером:
  - `POST /api/boardgames/invite`
  - `POST /api/boardgames/invite/update`
  - `POST /api/boardgames/invite/delete`

### WebSocket API

- Эндпоинт: `GET /ws/boardgames`
- Базовые события:
  - `auth`, `list_rooms`, `create_room`, `join_room`, `leave_room`
  - `start_game`, `move`, `place_bet`
  - серверные: `rooms_list`, `room_joined`, `room_updated`, `room_state`, `bet_ok`, `error`
- В `create_room` передаётся `botDifficulty`, в состоянии комнаты возвращается `botDifficulty`

### Покер: комнаты на обезьянки

- В настройках комнаты доступен режим `Monkeys → chips` (альтернатива обычному режиму `Chips only`)
- Курс фиксированный: `1 🐵 = 10 chips`
- Вход в такую комнату: обязательный buy-in, который равен `startChips / 10`
- `startChips` в monkey-режиме сервером приводится к кратному 10, чтобы курс всегда был точным
- При выходе из комнаты оставшиеся фишки конвертируются обратно в обезьянки по тому же курсу
- Смена валютного режима после создания комнаты запрещена (условия игры фиксируются при создании)

### Black Jack: комнаты на фишки и обезьянки

- Добавлен мультиплеерный Black Jack с комнатами (`/blackjack`) и WebSocket-эндпоинтом `/ws/blackjack`
- Формат комнаты такой же, как в покере: можно играть с ботами и с реальными игроками
- Поддерживаются 2 валютных режима:
  - `Chips only` — игра только на фишки
  - `Monkeys → chips` — buy-in обезьянками с курсом `1 🐵 = 10 chips`
- Настройки комнаты:
  - `startChips` — стартовый стек для каждого участника
  - `tableBet` — фиксированная ставка раздачи (списывается в начале раунда)
  - `botCount` — количество ботов (до 5)
- Действия игрока в раздаче: `hit`, `stand`, `double`
- Раунд завершается после ходов всех участников и авто-хода дилера (правило дилера: добор до 17)
- Выплаты:
  - победа: `x2` от ставки
  - push: возврат ставки
  - blackjack: `x2.5` от ставки (включая возврат)

## Принципы экономики

1. **Контролируемый приток** — фиксированный дневной бонус, RTP < 100%
2. **Мотивация возврата** — бонус по кнопке, streak-бонусы
3. **Ощущение прогресса** — баланс растёт, открываются новые возможности
4. **Азарт без риска** — виртуальная валюта, нулевые реальные ставки
5. **Баланс sink/faucet** — приток (бонусы) ≈ оттоку (RTP казино) для стабильности

## Правила оформления казино

### UI/UX
- Тёмная тема с яркими акцентами (градиенты на карточках игр)
- Баланс всегда видимо сверху экрана

- Карточки игр — 2 колонки, яркие градиенты, эмодзи, анимация свечения
- Недоступные игры — затемнены, с лейблом "Скоро"

## Метрики и статистика

Метрики казино собираются через **Prometheus** (MetricsEngine) — аналогично покеру и чат-статистике.

### Prometheus-счётчики

| Метрика | Лейблы | Описание |
|---|---|---|
| `casino_games_total` | `user_id`, `user_name`, `game`, `result` | result: `win` / `loss` |
| `casino_monkeys_won_total` | `user_id`, `user_name`, `game` | сумма выигрышей |
| `casino_monkeys_bet_total` | `user_id`, `user_name`, `game` | сумма ставок |
| `casino_bonus_total` | `user_id`, `user_name` | сумма полученных бонусов |

`game` ∈ `{slots, coinflip, roulette, slots5x5, rocket, race, chess, checkers}`

### API

Все эндпоинты авторизуются через HTTP-only cookie `casino_sid`, которая устанавливается при создании сессии. `user_id` не передаётся в URL.

| Метод | Эндпоинт | Описание |
|---|---|---|
| `POST` | `/api/casino/session` | Создание сессии: принимает `initData` (Telegram WebApp), валидирует HMAC, возвращает `{monkeys, lastBonusClaim, token}`, ставит cookie |
| `GET` | `/api/casino/balance` | Баланс и `lastBonusClaim` из БД |
| `POST` | `/api/casino/event` | Результат раунда: `{game, bet, win, token}` → валидация лимитов → обновление баланса в БД + `metrics.inc()` |
| `POST` | `/api/casino/bonus` | Бонус → проверяет кулдаун 24ч, обновляет баланс/время в БД + `metrics.inc()` |
| `GET` | `/api/casino/stats` | Сводка по Prometheus + `boardgames` (партии/победы) + `achievements` |

### Хранение в БД

| Поле `User` | Тип | Описание |
|---|---|---|
| `monkeys` | `int` | Текущий баланс обезьянок (default: 100) |
| `casino_last_bonus` | `float` | Unix-timestamp последнего бонуса |

Баланс виден в команде `/me` бота.

### Поток данных

1. Фронтенд при загрузке → `POST /api/casino/session` с `initData` → получает баланс, token, cookie
2. Фронтенд: после анимации раунда → `POST /api/casino/event` с `{game, bet, win, token}`
3. Бэкенд: валидирует token/лимиты/cooldown, обновляет `user.monkeys` в БД, вызывает `metrics.inc()`
4. Фронтенд: при возврате в хаб → `GET /api/casino/stats`
5. Бэкенд: PromQL-запросы к Prometheus → JSON-ответ

### UI

- Блок статистики отображается на хабе казино между бонусом и карточками игр
- Скрыт, если ещё ни одной игры не сыграно
- По умолчанию — компактная сводка (3 ячейки: Игр, Выиграно, Нетто)
- Кнопка «Подробнее ▼» раскрывает полную статистику: поставлено, бонус, винрейт, разбивка по играм
- Оформление: полупрозрачный тёмный фон, стиль как у PokerStatsBlock

## Безопасность и античит

### Аутентификация сессии

| Параметр | Значение |
|---|---|
| Метод | Telegram WebApp `initData` → HMAC-SHA256 валидация |
| Ключ HMAC | `HMAC(SHA256, "WebAppData", BOT_TOKEN)` |
| Сессия | HTTP-only cookie `casino_sid` (Secure, SameSite=Lax, path=/api/casino) |
| TTL сессии | 24 часа (`CASINO_SESSION_TTL = 86400`) |
| CSRF-токен | Одноразовый `token` (secrets.token_urlsafe) передаётся в каждый `/api/casino/event` |

### Серверная валидация раундов

| Проверка | Описание |
|---|---|
| Rate-limit | Минимум 1.5 сек между спинами (`CASINO_SPIN_COOLDOWN`) — HTTP 429 при нарушении |
| Max bet | Лимит ставки по играм: slots=10, coinflip=50, roulette=50, slots5x5=10, rocket=50 |
| Max win | Лимит выигрыша по играм: slots=1500, coinflip=95, roulette=1800, slots5x5=5000, rocket=5000 |
| Баланс | Сервер проверяет `bet <= user.monkeys` перед списанием |
| Token | Сессионный token сверяется при каждом event-запросе |
| Game ID | Только `{slots, coinflip, roulette, slots5x5, rocket}` для event; race через отдельные эндпоинты |

### Генерация случайных чисел

На фронтенде используется `crypto.getRandomValues()` (Web Crypto API) вместо `Math.random()` для устойчивости к предсказанию.

```js
function secureRandom() {
  const buf = new Uint32Array(1)
  crypto.getRandomValues(buf)
  return buf[0] / 0x100000000
}
```

## Звуковая система

Все звуки казино генерируются процедурно через **Web Audio API** — без аудиофайлов.

### Архитектура

- Единый `AudioContext` (lazy-init при первом взаимодействии)
- Warm-up при первом `pointerdown` для обхода autoplay-политики браузеров
- Reverb через конвольвер с процедурным IR-буфером (кешируется)
- Mute/unmute с сохранением в `localStorage` (`casino_muted`)

### Звуковые события

| Звук | Когда играет |
|---|---|
| `spin` | Начало вращения барабанов (sweep + noise + kick) |
| `reelStop` | Остановка каждого барабана (kick + tones) |
| `win` | Обычный выигрыш (восходящая мелодия + аккорд + reverb) |
| `bigWin` | Крупный выигрыш (фанфара + двойной kick + sparkle-tones) |
| `lose` | Проигрыш (нисходящая мелодия) |
| `coinFlip` | Подбрасывание монетки (серия высокочастотных тонов с замедлением) |
| `coinLand` | Приземление монетки (kick + металлические обертоны) |
| `rouletteSpin` | Вращение рулетки (sweep + noise + ритмичные клики) |
| `rouletteBall` | Шарик падает в ячейку (kick + нисходящие тоны) |
| `bonus` | Получение дневного бонуса (фанфара + аккорды + sparkle) |
| `tick` | Выбор ставки в рулетке / размещение ставки в ракетке (короткий клик) |
| `rocketLaunch` | Запуск ракеты (rumble + whoosh + rising tones) |
| `rocketCrash` | Взрыв ракеты (kicks + noise + descending sweeps) |

### Эффекты

- **Reverb**: конвольвер с decay 0.8–2.5 сек (зависит от типа звука)
- **Dry/Wet mix**: 70% dry, 35% wet
- **Компоненты**: `tone()`, `sweep()`, `noise()`, `chord()`, `kick()` — комбинируются для создания звуков
